H.1. Testing and Installing the user library

To test the library before installing it as the operational version, build the library using libmake, then set and export the FPA_SHLIB_PATH variable to the directory in which the library was built. Running any FPA application from that environment will access the newly built library.

Once tested, you may then install the library in the standard directory, using the libput script. The libput script will fail if the shared library is in use, so be sure to shutdown the depiction editor and ingest daemon before you transfer the files. Unset the FPA_SHLIB_PATH variable and test that the library was transferred properly. The librestore script provides the ability to go back to the last working version of the library, if it turns out to have been installed prematurely!
